# Repository Cleanup Plan

The public repository should look like a focused portfolio project.

## Keep

- Application code.
- Tests and CI configuration.
- English README, quickstart, security, and contribution docs.
- Clean screenshots of the app only.
- Policies that explain architecture and safety boundaries.

## Exclude

- Local credentials.
- Sessions.
- Private memories.
- Backups and exports.
- Browser screenshots that show unrelated tabs, taskbars, or personal work context.

## Done definition

A reviewer should understand the project purpose, run it locally, inspect tests, and see a clean UI without private data exposure.
